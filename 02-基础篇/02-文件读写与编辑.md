# 文件读写与编辑

Claude Code 操作文件靠三个核心工具，理解它们的分工能帮你避开大部分"改坏文件"的坑。

## 核心三件套

### Read——读文件

读取文件内容，支持行号范围、PDF 分页、图片识别。Claude 在修改任何文件之前都会先 Read 一遍，确认当前内容。

```
Read /src/utils.ts           # 读整个文件
Read /src/utils.ts 10-50     # 只读第 10-50 行
Read /docs/spec.pdf 1-5      # 读 PDF 前 5 页
```

### Edit——精确替换

找到文件中的一段文本，精确替换成新内容。产生的 diff 最小，不会误伤其他代码。

```
Edit /src/auth.ts
  old: "const timeout = 3000"
  new: "const timeout = 5000"
```

### Write——整文件写入

把整个文件内容一次性写入。仅用于新建文件或需要大幅重写的场景。

> [!warning]
> Write 会覆盖整个文件。如果只是改几行，用 Edit 更安全。

## 什么时候选哪个

| 场景 | 推荐工具 | 原因 |
|------|----------|------|
| 改 1-3 行 | Edit | diff 最小，不会误伤 |
| 改大段逻辑 / 新建文件 | Write | Edit 要匹配的文本太长反而容易失败 |
| 不确定改完对不对 | 先 Read 再 Edit | 看清现状再动手 |
| 重命名变量（多处出现） | Edit + `replace_all` | 一次替换所有出现 |

## NotebookEdit：处理 .ipynb

Jupyter Notebook 是按 cell 组织的，Claude 用 `NotebookEdit` 工具按 cell 编辑：

- 替换某个 cell 的内容
- 插入新 cell
- 删除 cell

不需要你手动处理 JSON 结构，直接说"把第三个 cell 的绘图代码改成用 seaborn"就行。

## Glob / Grep：找文件、搜内容

在动手改代码之前，Claude 经常需要先找到目标文件：

- **Glob**——按文件名模式匹配：`*.tsx`、`src/**/test_*.py`
- **Grep**——按内容搜索：找所有包含 `deprecated` 的文件

这两个比在 Bash 里跑 `find` / `grep` 更高效，因为它们是专用工具，结果结构化且不会被 shell 转义问题坑到。

> [!tip]
> 如果你想让 Claude 了解项目结构，与其说"看一下目录"，不如说"用 Glob 找所有 `.ts` 文件"——它能更快建立全局认知。

## 常见坑

**1. "重写整个文件"容易漏行**

让 Claude 用 Write 重写一个 500 行的文件，它可能漏掉某些边界处理或注释。更好的做法是让它用 Edit 只改需要改的部分。

```
❌ "把 src/api.ts 重写一下，加上错误处理"
✅ "给 src/api.ts 里的每个 fetch 调用加上 try-catch"
```

**2. Edit 失败反复重试**

Edit 要求 `old_string` 在文件中唯一匹配。如果匹配到多处或零处，就会失败。Claude 有时会反复重试同样的操作——这时候你介入一下，告诉它具体改哪个位置（比如"改第 42 行那个"）。

**3. 大文件一次写入被截断**

超过 5000 token 的文件一次 Write 可能被截断。Claude 应该分批写入，但如果你发现文件不完整，提醒它"文件被截断了，继续写完"。
