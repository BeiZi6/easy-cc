# 界面与基本概念

启动 `claude` 之后，你面对的是一个看起来很简单的终端界面。但它藏了不少约定，搞清楚这些能让你后面少踩坑。

## 输入框前缀

Claude Code 的输入框不只是打字聊天，不同的前缀触发不同行为：

| 前缀    | 作用                                                     | 示例                          |
| ----- | ------------------------------------------------------ | --------------------------- |
| （无前缀） | 普通对话，发给 Claude 处理![[assets/截屏2026-05-21 20.07.44.png]]        | `帮我重构这个函数`                  |
| `/`   | Slash 命令，触发内置功能![[assets/截屏2026-05-21 20.06.54.png]]          | `/compact`、`/clear`、`/exit` |
| `!`   | 直接执行 shell 命令，输出会进入对话上下文![[assets/截屏2026-05-21 20.08.03.png]] | `!git status`               |
| `#`   | 写入 Memory（CLAUDE.md）![[assets/截屏2026-05-21 20.06.24.png]]     | `#这个项目用 pnpm 不用 npm`        |
| `@`   | 引用文件或 mention agent![[assets/截屏2026-05-21 20.08.18.png]]      | `@src/utils.ts 这个文件有 bug`   |

> [!tip] `!` 前缀的妙用
> 用 `!` 跑命令的输出会自动成为对话的一部分。比如你跑 `!cat error.log`，Claude 就能直接看到日志内容并帮你分析，不用手动复制粘贴。

详细的 slash 命令列表见 [[../02-基础篇/04-Slash命令一览]]。

## 状态指示

对话过程中，终端会展示几种状态信息：

**Thinking 动画**：Claude 在思考时会有一个旋转的加载指示，表示模型正在生成回复。![[assets/截屏2026-05-21 20.09.02.png]]

**Tool Use 卡片**：当 Claude 要执行操作（读文件、写文件、跑命令）时，会展示一个折叠的工具调用块，里面有操作类型、目标路径、具体内容。你可以展开查看细节，再决定批准还是拒绝。

**Token 用量**：对话进行中会显示当前 session 的 token 消耗。留意这个数字——上下文窗口是有限的，塞满了就需要 compact。![[assets/截屏2026-05-21 20.09.36.png]]

## 三种工作模式

Claude Code 有三种模式，控制它的"自主程度"：

**默认模式（Supervised）**

每个工具调用都会暂停等你确认。适合新手、敏感操作、不熟悉的项目。![[assets/截屏2026-05-21 20.10.14.png]]

**Auto 模式**

低风险操作（读文件、写代码）自动放行，只有高风险操作（删文件、跑未知命令）才会问你。适合你已经信任它的判断、想要流畅体验的时候。![[assets/截屏2026-05-21 20.09.59.png]]

```
/auto
```

**Plan 模式**

Claude 只输出计划，不执行任何操作。你审完计划说"行"，它才开始干活。适合大型任务、你想先看全貌再动手的场景。![[assets/截屏2026-05-21 20.10.36.png]]

```
/plan
```

> [!warning] 模式选择建议
> 刚上手先用默认模式，建立对 Claude 行为边界的直觉。等你能预判它下一步要干嘛了，再切 Auto。Plan 模式留给跨文件的大改动。
> 除了这三种基本模式外，还有yolo模式bypass permissions on ，这时候Claude code的所有命令都不需要你来进行审批![[assets/截屏2026-05-21 20.11.23.png]]

详见 [[../02-基础篇/05-Plan模式与Auto模式]]。

## 对话状态管理

一个 session 聊久了，上下文会越来越大。Claude Code 提供几个命令来管理：

| 命令 | 作用 |
|---|---|
| `/clear` | 清空当前对话，从零开始 |
| `/compact` | 压缩上下文，保留关键信息，释放 token 空间 |
| `/resume` | 恢复上一次的对话（在同一目录重新启动后可用） |

当你看到 token 用量接近上限，或者 Claude 开始"忘事"，就是该 `/compact` 的时候了。

Claude Code 还支持后台运行：你可以启动一个任务后让它在后台跑，自己去干别的。这在 CI 场景和长时间任务里很有用。

详见 [[../02-基础篇/06-上下文管理-clear-与-compact]]。

## 关键术语速记

后面的章节会反复出现这些概念，先混个脸熟：

| 术语 | 一句话解释 |
|---|---|
| Tool / Tool Use | Claude 执行操作的方式——读文件、写文件、跑命令都是"工具调用" |
| Tool Result | 工具执行后的返回结果，会回传给 Claude 作为下一步的输入 |
| Context Window | 一次对话能装多少内容的上限，超了就得 compact |
| Compaction | 把长对话压缩成摘要，腾出空间继续聊 |
| Subagent | Claude 派出去干活的"分身"，处理独立子任务 |
| Hook | 在特定事件（如工具调用前后）自动触发的脚本 |
| Skill | 可加载的能力包，给 Claude 特定领域的知识和行为模式 |
| MCP | Model Context Protocol，让 Claude 连接外部服务的协议 |
| Permission | 权限控制，决定 Claude 能做什么不能做什么 |
| Sandbox | 沙箱环境，限制 Claude 的操作范围 |
| Worktree | Git worktree，让 Claude 在隔离分支上干活不影响主分支 |
| Session | 一次完整的对话，从启动到退出 |

> [!tip] 不用背
> 这些概念用到的时候自然就懂了。现在只需要知道"有这么个东西"，后面每个都有专门章节展开。
