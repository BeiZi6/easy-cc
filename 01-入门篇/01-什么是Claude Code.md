# 什么是 Claude Code

> 一句话：Claude Code 是 Anthropic 官方推出的「住在终端里的编程 Agent」。

## 它是什么

Claude Code 是一个跑在终端里的 CLI 工具。装好以后在任意项目根目录敲一句 `claude`，就进入一个会话。

它的工作方式很朴素：

- 你用自然语言提需求；
- 它自己决定要读哪些文件、跑哪些命令、改哪些代码；
- 改完跑测试，看到报错就继续修，直到任务完成或者它觉得需要你拍板。

它的本体是 Anthropic 官方维护的 npm 包 `@anthropic-ai/claude-code`，模型是 Claude 系列（Opus / Sonnet / Haiku）。除了 CLI，还有同一个内核驱动的桌面 App、Web 版（claude.ai/code）和 IDE 插件，但**终端是一等公民**——文档、Skills、Hooks、MCP 这些扩展机制都是围绕 CLI 设计的。

## 它不是什么

容易混淆的几样东西，先撇清：

- **不是 IDE 插件**。不依赖 VS Code/JetBrains，SSH 进服务器、Docker 里、CI runner 里都能跑。
- **不是浏览器里的聊天框**。它会真实地读写你硬盘上的文件、执行 shell 命令。
- **不是 ChatGPT 套壳**。模型是 Claude，工具调用、上下文管理、权限模型都是 Anthropic 自己的一套。
- **不是补全引擎**。它不在你打字时插建议，而是接到一句需求后**自己规划并执行整段工作**。

## 和别的工具有什么区别

我想这部分本不用介绍，只做内容完整性设计。

| 对比对象 | 关键差异 |
|---|---|
| ChatGPT 网页版 | 它能直接动你电脑上的文件、跑命令；网页版只能复制粘贴 |
| GitHub Copilot | Copilot 偏「行级补全」，Claude Code 偏「整段任务的 Agent」 |
| Cursor / Windsurf | Cursor 把 Agent 嵌进 IDE；Claude Code 反过来——以 CLI 为主，IDE 为辅，更适合服务器和自动化场景 |
| Aider / Continue | 同为 CLI 编程 Agent，但 Claude Code 是模型厂自家产品，新模型、新能力（Skills、Subagent、Memory）通常先在这里落地 |
| Cline / Roo Code | 都是开源 Agent，Claude Code 闭源但配套生态（MCP、Skills、Hooks）更完整 |

> [!tip] 选型提示
> 如果你已经离不开 IDE 内嵌体验，Cursor 更顺手；如果你愿意把工作迁到终端、追求自动化和无人值守，Claude Code 上限更高。

## 它能做什么

下面五个是上手以后会高频用到的场景：

1. **一句话需求 → 跨文件改代码**
   "把 `User` 模型加一个 `last_login_at` 字段，迁移、ORM、API、测试一起改。"
2. **跑测试 → 看输出 → 自我修正**
   它会执行 `npm test`、读报错、定位文件、改代码，循环直到测试绿。
3. **翻仓库、写文档**
   "看一下这个项目的鉴权流程，画个时序图，写到 `docs/auth.md`。"
4. **跑批处理脚本**
   把一堆 Markdown 文件统一改格式、把日志按规则切分归档、批量重命名图片。
5. **CI 里自动审 PR / triage Issue**
   配 GitHub Actions，让它在 PR 上留评论、自动给 issue 打标签、关联重复单。详见 [[../07-自动化与CI/02-自动PR审查]]。

## 它的局限

不是无敌的，下面这几类活儿它现在还吃不消，或者需要你盯紧：

- **超长任务**：上下文窗口虽然大，但跨几十轮工具调用后会需要 `/compact`，关键状态可能丢。
- **大型架构改造**：动到几十个文件、跨模块语义的重构，建议先让它写 spec/plan，人审过再放手干。
- **强外部依赖的调试**：网络抖动、第三方服务异常，它没法"亲自"复现，需要你喂日志。
- **创造性的产品决策**：要不要做这个功能、UI 走哪种风格——这类问题它会给建议，但拍板还是人的事。
- **成本不可见的烧 token**：开 Auto 模式跑一晚上，账单可能很惊人，详见 [[../08-避坑与反模式/03-Token成本控制]]。

## 一句话收尾

> 把 Claude Code 当成一个「能动手、有判断、需要被管教」的初级到中级工程师，比把它当成"更聪明的搜索框"更接近真相。
