# 安装与登录

> 目标：在你的机器上装好 Claude Code，能跑起来 `claude --version`，能进入一个会话。![[assets/截屏2026-05-21 20.08.36.png]]

## 环境要求

- **Node.js**：18+，推荐 LTS（20 或 22）。`node -v` 查一下，太旧会装不上。
- **操作系统**：
  - macOS / Linux：原生支持，开箱即用。
  - Windows：必须走 **WSL2**，原生 PowerShell/CMD 不在官方支持列表里。
- **终端**：任意现代终端，建议支持 256 色 + Unicode（iTerm2、WezTerm、Alacritty、Windows Terminal 都行）。

> [!tip] Node 用什么管
> 别用系统包管理器装的 Node（容易触发权限问题），用 [nvm](https://github.com/nvm-sh/nvm) 或 [fnm](https://github.com/Schniz/fnm) 装到用户目录最省事。

## 安装方式

### npm 全局安装（推荐）

```bash
npm install -g @anthropic-ai/claude-code
```

验证：

```bash
claude --version
# 应输出类似 2.x.y
```

升级：

```bash
claude update
# 或者重装最新版
npm install -g @anthropic-ai/claude-code@latest
```

### 不想全局装

也可以 `npx @anthropic-ai/claude-code`，每次现拉。代价是首次启动慢，且没法享受 `claude` 这种短命令。

### 常见安装问题

- **`command not found: claude`**：npm 全局 bin 不在 `PATH` 里。`npm config get prefix` 看路径，把 `<prefix>/bin` 加到 `PATH`。
- **`EACCES` 权限错误**：别用 `sudo npm install -g`。改用 nvm/fnm 管 Node，或者把 npm 全局目录指到用户家目录。
- **公司代理证书报错**：在内部 PKI 环境下，`export NODE_EXTRA_CA_CERTS=/path/to/corp-ca.pem`。

## 登录方式

四种登录路径，按使用场景从轻到重：

### 1. Pro / Max / Team 订阅登录（最常见）

进入会话后跑 `/login`，浏览器弹一个 OAuth 授权页，登录 Anthropic 账号即可。token 存在本地 `~/.claude/` 下，下次自动续。

适合：日常自己用、有 Claude 订阅的人。

### 2. 直接用 Anthropic API Key

```bash
export ANTHROPIC_API_KEY=sk-ant-xxx
claude
```

适合：想精确控制 token 成本、跑无人值守脚本、CI 环境。

### 3. Bedrock / Vertex 企业接入

```bash
# AWS Bedrock
export CLAUDE_CODE_USE_BEDROCK=1
export AWS_REGION=us-east-1

# Google Vertex
export CLAUDE_CODE_USE_VERTEX=1
export CLOUD_ML_REGION=us-east5
export ANTHROPIC_VERTEX_PROJECT_ID=your-gcp-project
```

适合：公司有 AWS/GCP 框架协议、走云厂商账单的团队。

### 4. 中转站 API（baseurl + API-key）

社区有不少把 Claude API 转成兼容端点的中转服务，通过 `ANTHROPIC_BASE_URL` 指向中转地址：

```bash
export ANTHROPIC_BASE_URL=https://your-proxy.example.com
export ANTHROPIC_API_KEY=your-proxy-key
claude
```

> [!warning] 中转的风险
> 中转服务能看到你所有 prompt 和代码片段，等于把仓库内容交给第三方。涉密/商业项目千万别走中转。
> 个人学习、开源项目可以接受，但要清楚账号风控、计费透明度都在中转方手里。

环境变量推荐写到 `~/.zshrc` 或 `~/.bashrc`，重开终端后生效。

## 第一次启动

在**项目根目录**敲 `claude`：

```bash
cd ~/code/my-project
claude
```

首次启动会问几件事：

1. **信任此目录？** Claude Code 默认对每个目录是「不信任」状态，第一次进入会让你确认。说 yes 之后会在 `.claude/settings.local.json` 里记一下。
2. **要不要建立 `CLAUDE.md`？** 项目级的指令文件，可以先跳过，之后用 `/init` 自动生成（详见 [[../04-配置与定制/02-CLAUDE.md分层机制]]）。
3. **选模型。** 默认是当前最新的 Sonnet/Opus，可以 `/model` 切换。

到这一步就能正常对话了，下一节 [[03-第一次对话]] 跑一个完整的 Hello World。
