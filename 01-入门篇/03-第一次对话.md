# 第一次对话

> 目标：在 5 分钟内跑通"让 Claude 改一个文件"的最小循环。

## 建一个测试目录

找个干净的地方开始，别在你的正式项目里试——万一手滑批准了什么奇怪的操作：

```bash
mkdir hello-cc && cd hello-cc
```

这个目录就是 Claude Code 的"工作区"。它只会在当前目录及子目录里读写文件，不会跑到别处乱动。

## 启动

```bash
claude
```

看到一个 `>` 提示符（或带颜色的输入框），就说明启动成功了。第一次启动可能会让你选模型或确认账户，跟着走就行。

> [!tip] 如果报错 `command not found`
> 回头看 [[02-安装与登录]]，确认 `npm install -g @anthropic-ai/claude-code` 跑过了，且 npm 全局 bin 在你的 PATH 里。

## 第一个任务

在提示符里输入：

```
写一个 Python 脚本，输出当天日期，文件名叫 today.py
```

按回车，然后观察。

## 观察它做了什么

Claude 收到你的需求后，通常会经历这几步：

1. **规划**（如果开了 Plan 模式）——它会先说"我打算创建一个文件 `today.py`，内容是……"
2. **调用 Write 工具**——你会看到一个工具调用卡片，显示它要写入的文件路径和内容
3. **验证**（可选）——它可能会用 Bash 工具跑一下 `python today.py`，确认输出正确

每一步都会暂停等你确认，不会偷偷执行。

## 批准或拒绝工具调用

当 Claude 想执行一个操作时，终端会弹出确认提示：

- **y**（或直接回车）：批准，让它执行
- **n**：拒绝，它会换个思路或者问你怎么办
- **编辑权限**：你可以选择"始终允许这类操作"，以后同类调用就不再问了

> [!warning] 第一次先别开 Auto 模式
> 新手阶段建议每个操作都过一眼，建立对它行为边界的直觉。等你熟悉了再开 Auto 放飞。

## 追问改进

文件创建成功后，继续在同一个会话里说：

```
加一个命令行参数 --timezone，可以指定时区输出对应时间
```

观察它怎么做：

- 它会读取刚才创建的 `today.py`
- 修改代码，加上 `argparse` 或 `sys.argv` 的处理
- 可能再跑一次验证

这就是 Claude Code 的核心循环：**你提需求 → 它改代码 → 你确认 → 继续迭代**。

## 退出

聊完了，两种方式退出：

- `Ctrl+D`
- 输入 `/exit`

会话历史会保存，下次在同一目录启动 `claude` 可以用 `/resume` 接着聊。

## 复盘：刚才发生了什么

回顾一下这 5 分钟里的完整流程：

1. 你用自然语言描述了需求
2. Claude 把需求拆解成具体的文件操作和命令
3. 每个操作都经过你的批准才执行
4. 你可以在同一个会话里持续迭代，它记得上下文

## 心智模型

把 Claude Code 想成一个：**能看你屏幕、能动你文件、但每个动作要你点头的远程结对伙伴**。

它不是一个"问答机器人"——它是一个有手的 Agent。你说"加个字段"，它真的会去改文件、跑迁移、跑测试。区别只在于，它每一步都会停下来等你说"行，干吧"。
