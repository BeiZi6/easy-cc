# 上下文污染

> 「你跟它聊得越久，它越糊」——这是用户最常报"模型变笨"的原因。

## 三种典型污染

### 1. 历史里塞了大量过期信息
- 删过的代码片段、回退的方案、半完成的尝试
- 模型基于这些"幽灵上下文"做决策

### 2. 反复纠错叠加
- 你纠正一次 → 它回应 → 不对 → 你再纠正
- 每轮都增加新词；模型可能锁死在错误轨道上

### 3. 任务切换不清场
- 早上改前端，下午直接让它"修一下后端 bug"
- 它还在前端 mindset 里

## 解药

- **超过 2 次纠错** → `/clear` 重开
- **任务切换** → `/clear` 或新开 session
- **走偏了，但要保留主线** → `/compact <focus>` 局部压缩
- **小问题不入历史** → `/btw 这个错误信息什么意思？`（2026 新）
- **关键发现 → 落盘**（plan.md / progress.md），下次让它读文件
- **重要 session 用 `/rewind`**（Esc Esc）回到 checkpoint
