# 术语表

> 按字母顺序。每条尽量一句话讲清。

## A
- **Agent** —— 能调工具、跑循环的 LLM。Claude Code 本身就是一个 agent。
- **Agent Teams** —— 2026 新功能，多个 session 自动协调。
- **Agent View** —— 一屏监控多 session。
- **Auto Memory** —— Claude 自动累积学习的记忆系统。
- **Auto Mode** —— 用 Sonnet 4.6 当 classifier 自动放行低风险操作。

## B
- **Bedrock** —— AWS 上的 Claude 接入入口。

## C
- **CLAUDE.md** —— 项目/用户级"项目说明书"，自动注入上下文。
- **Channels** —— Telegram/Discord/iMessage 推消息进 Claude 的功能。
- **Checkpoint** —— Claude 自己打的代码快照，可 `/rewind` 回退。
- **Compaction** —— 上下文压缩，保留摘要丢细节。
- **Connector Directory** —— 官方审核过的 MCP 列表。
- **Context Window** —— 模型一次能看到的总长度（token 数）。
- **Context Engineering** —— 让对的东西进上下文、错的东西出去的方法论。

## H
- **Headless 模式** —— 非交互模式，`claude -p`，CI 必备。
- **Hook** —— 工具调用/会话事件触发的脚本。

## M
- **MCP** —— Model Context Protocol，LLM ↔ 工具的标准协议。
- **MCP App** —— 带 UI 的 MCP（2026 新）。
- **Memory** —— Claude 的持久记忆系统。

## O
- **Orchestrator-Worker** —— 一个主 agent 派任务给多个 worker agent。
- **Output Style** —— 控制回复风格（Concise / Explanatory / 自定义）。

## P
- **Permission Mode** —— default / acceptEdits / plan / auto / bypassPermissions / dontAsk。
- **Plan 模式** —— 只规划不动手，写计划等用户批准。
- **Plugin** —— 把 skills + hooks + subagents + MCP 打包分发的格式。

## R
- **Ralph Loop** —— Geoffrey Huntley 倡导的"一个 while-loop 跑同一个 Claude"范式。
- **Routines** —— 云端定时任务，电脑关机也跑。

## S
- **Sandbox** —— OS 级隔离，限制文件系统/网络访问。
- **Session** —— 一次对话；可命名、continue、resume。
- **Showboat 模式** —— 让 agent 用 HTML 输出报告（Simon Willison 命名）。
- **Skill** —— 按需加载的能力包，比 slash 大、比 subagent 小。
- **Slash 命令** —— 输入框 `/` 开头的内置/自定义命令。
- **Slopfork** —— Armin Ronacher 命名：AI 改出"看起来像但不是原作"的库。
- **Status Line** —— 终端底部状态栏，可自定义。
- **Subagent** —— 由主 agent 委托的隔离 agent。

## T
- **TDD** —— 测试驱动开发，对 LLM 协作特别有效。
- **Tool Use** —— LLM 调用工具的过程。

## V
- **Vertex AI** —— GCP 上的 Claude 接入入口。

## W
- **Worktree** —— git worktree，物理隔离的工作目录，给并行 Claude 用。
- **Writer / Reviewer 双 session** —— 一个 session 写、另一个全新 context 评审。
