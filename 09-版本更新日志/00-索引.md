# 09 版本更新日志

> Claude Code 大约每周都有更新。这里记录**对使用方式有影响**的关键变更。
> 不是 changelog 镜像——只挑会改变你日常用法的事。

## 维护规则

每条记录三件事：
1. **日期 + 版本号**（如已知）
2. **变更点 + 原因**（猜不准就只写"变化"）
3. **受影响章节**：用 wiki link 指过去，并去那个章节顶部加 `> [!warning] 自 vX 起 ...` 提示

模板：

```markdown
## YYYY-MM-DD · vX.Y.Z

- 新增：xxx
- 变更：xxx
- 弃用：xxx
- 受影响章节：[[../04-配置与定制/04-Hooks钩子系统]]
```

## 近期版本变更（v2.1.128 → v2.1.150）

### 2026-05-23 · v2.1.149 ~ v2.1.150

- v2.1.150：内部基础设施改进，无面向用户的变更
- 新增：`/usage` 显示额度用量分类明细（skills / subagents / plugins / 各 MCP server）
- 新增：`/diff` 详情视图支持键盘滚动（方向键 / j/k / PgUp/PgDn / Space / Home/End）
- 新增：Markdown 输出渲染 GFM 任务列表复选框（`- [ ]` / `- [x]`）
- 新增（企业）：`allowAllClaudeAiMcps` 托管设置
- 修复：PowerShell `cd` 权限绕过
- 受影响章节：[[../02-基础篇/04-Slash命令一览]]、[[../08-避坑与反模式/03-Token成本控制]]

### 2026-05-22 · v2.1.147 ~ v2.1.148

- 变更：`/simplify` 更名为 `/code-review`，支持 effort 级别和 `--comment` 发 PR 行内评论
- 新增：`Ctrl+T` 在 `claude agents` 中固定后台 session（pinned），idle 时不被回收
- 修复：v2.1.148 热修 Bash 工具返回 exit code 127 的回退问题
- 受影响章节：[[../02-基础篇/04-Slash命令一览]]

### 2026-05-14 · v2.1.144 ~ v2.1.145

- 修复：macOS Full Disk Access 保护目录下后台 session 崩溃
- 修复：`head`/`tail` 现在满足 read-before-edit 检查；`egrep`/`fgrep`/`git grep` 的"无匹配"不再报错
- 修复：`/branch` 在 worktree 或后台 session 中报 "No conversation to branch"
- 改进：auto-updater 重试网络失败、报告具体错误分类和 OS 错误码
- 改进：大文件编辑的 diff 渲染性能

### 2026-05-11 · v2.1.136 ~ v2.1.143

- 新增：`claude agents` Agent View——一屏监控所有 Claude Code session
- 新增：`/goal` 命令——设定完成条件，Claude 跨轮次持续工作
- 新增：Fast 模式默认使用 Opus 4.7
- 新增：Rewind 菜单 "Summarize up to here"——选择性压缩早期上下文
- 新增：`settings.autoMode.hard_deny` 无条件拦截规则
- 新增：Hooks / Bash 收到 `$CLAUDE_EFFORT` 环境变量（v2.1.133）
- 受影响章节：[[../02-基础篇/05-Plan模式与Auto模式]]、[[../02-基础篇/06-上下文管理-clear-与-compact]]、[[../04-配置与定制/04-Hooks钩子系统]]

### 2026-05-04 · v2.1.128 ~ v2.1.133

- 新增：Plugin 支持从 `.zip` 和 URL 加载
- 新增：`Ctrl+R` 跨项目搜索命令历史
- 新增：`worktree.baseRef` 设置（`fresh` | `head`）
- 新增：`sandbox.bwrapPath` / `sandbox.socatPath` 托管设置（Linux/WSL）
- 修复：并行 session 401 refresh-token 竞态
- 修复：MCP OAuth 不遵守 HTTP(S)_PROXY / NO_PROXY / mTLS
- 受影响章节：[[../06-高阶篇/03-Git-Worktree并行实例]]、[[../04-配置与定制/01-settings.json三级配置]]
