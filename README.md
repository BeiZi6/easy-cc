# Claude Code 折腾指南

> 一份从零基础到高阶玩法的 Claude Code 中文指南。
> Claude Code 更新极快，本仓库按"框架 + 增量更新"的方式维护。

## 说明

这不是官方文档的翻译，就是我自己用 Claude Code 过程中顺手记下来的东西。官方文档写得挺全，但很多细节要自己踩过才知道——比如 context 什么时候会悄悄污染、hooks 的执行顺序为什么跟你想的不一样、MCP 连不上时该从哪里查。这些"踩坑笔记"散在各处很难找，所以整理成了这份指南。

内容按 Obsidian Vault 组织。用 Obsidian 打开体验最好，双向链接和图谱都能用。在 GitHub 或 VS Code 里直接看也没问题，`[[wiki-link]]` 显示成纯文本，不影响阅读。

Claude Code 更新很快，基本每周都有变化。我会在 [[09-版本更新日志]] 里追更，改动比较大的章节顶部会加提示。如果发现哪里写错了或者已经过时，欢迎提 Issue。

## 维护方式

Claude Code 大约每周都有更新。建议：

1. 每次官方发版后，在 [[09-版本更新日志]] 新增一条记录。
2. 受影响的章节顶部加 `> [!warning] 自 vX.Y 起行为变化` 说明。
3. 旧写法保留在原位，加 `~~删除线~~` 或 `> [!info] 历史写法` 折叠，方便回看。
4. 每章末尾的 "最后更新" 字段同步刷新。

> [!tip] 如何贡献
> 新增内容请遵循"先骨架后血肉"原则——先在对应章节列要点，确认结构后再展开。这能避免反复重写。
