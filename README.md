# Claude Code 折腾指南

> 一份从零基础到高阶玩法的 Claude Code 中文指南。
> Claude Code 更新极快，本仓库按"框架 + 增量更新"的方式维护。

## 说明

这不是官方文档的翻译，就是我自己用 Claude Code 过程中顺手记下来的东西。官方文档写得挺全，但很多细节要自己踩过才知道——比如 context 什么时候会悄悄污染、hooks 的执行顺序为什么跟你想的不一样、MCP 连不上时该从哪里查。这些"踩坑笔记"散在各处很难找，所以整理成了这份指南。

内容按 Obsidian Vault 组织，但在 GitHub 上直接读也没问题。如果你有 Obsidian，用它打开体验更好（双向链接、图谱都能用），见下方说明。

Claude Code 更新很快，基本每周都有变化。我会持续追更，改动比较大的章节顶部会加提示。如果发现哪里写错了或者已经过时，欢迎提 Issue。

## 用 Obsidian 打开（可选，体验更好）

强烈推荐用 [Obsidian](https://obsidian.md/) 阅读，wiki 链接、反向链接、图谱视图、callout 渲染全部可用。

```bash
git clone https://github.com/BeiZi6/easy-cc.git
```

clone 完成后，打开 Obsidian → "Open folder as vault" → 选中 `easy-cc` 目录即可。

## 维护方式

Claude Code 大约每周都有更新。建议：

1. 每次官方发版后，在 [09 版本更新日志](09-版本更新日志/00-索引.md) 新增一条记录。
2. 受影响的章节顶部加警告说明。
3. 旧写法保留在原位，加删除线或折叠，方便回看。
4. 每章末尾的"最后更新"字段同步刷新。

> 新增内容请遵循"先骨架后血肉"原则——先在对应章节列要点，确认结构后再展开。这能避免反复重写。
